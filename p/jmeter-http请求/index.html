2024/04/03

HTTP请求

JMeter提供多种取样器，HTTP取样器（HTTP Sampler）无疑是最核心的一个。HTTP取样器是用来模拟客户端和服务器之间HTTP请求和接收响应的工具，它可以发送各种类型的HTTP请求，如GET、POST、PUT、DELETE等，用于请求不同的资源。

HTTP取样器控制面板中包含元素：

  • 名称、注释

  • web服务器

  • HTTP请求

  • 参数

  • 消息体数据

  • 文件上传

  • 客户端实现

  • 超时

  • 从HTML文件获取所有内含的资源

  • 源地址

  • 代理服务器

  • 其他任务


Part1.web服务器&&HTTP请求

web服务器没什么好讲的，只保证IP/域名、端口、路径、请求方式、协议正确就好。内容编码最好输入UTF-8(兼容大部分语言)，防止接口解析时乱码。


自动重定向和跟随重定向

自动重定向只针对Get和Head请求，自动重定向转向到最终目标页面，但是Jmeter不记录重定向的中间页面过程，只记录最终页面返回结果。在结果树中，只能看到最终页面的服务器返回。

跟随重定向是http request 取样器的默认选项，会自动跳转到目标地址。Jmeter会记录重定向过程中的所有请求的响应结果。在结果树中，可以看到所有请求的服务器返回。

两者只能选中一个。这次我们访问知乎来进行测试。

示例Jmeter脚本

  • 测试计划中添加线程组

  • 线程组中添加HTTP取样器

    请求地址：HTTPS://www.zhihu.com

    请求方式：GET

  • 测试计划中添加查看结果树

勾选跟随重定向，运行结果

HTTP请求-0的GET https（请求地址）为//www.zhihu.com/，HTTP请求-1的GET https（请求地址）为//www.zhihu.com/signin?next=%2F。

HTTP请求-0的响应数据是一个重定向的a链接，链接地址为//www.zhihu.com/signin?next=%2F-即HTTP请求-1的请求地址。HTTP请求-1的响应才是最终的页面。Jmeter的查看结果树记录了重定向过程。

勾选自动重定向，运行结果

结果树中只有一个HTTP请求，请求地址为//www.zhihu.com/signin?next=%2F-即最终的页面数据。查看结果树中没有记录重定向过程。


使用KeepAlive

选中该选项，JMeter和目标服务器之间使用Keep-Alive方式进行HTTP通信，默认选中。keep-alive属性，用于建立长连接，可以避免连接频繁建立和释放的开销。有兴趣的可以搜索HTTP的Keep-Alive模式，详细了解。

Part2.请求参数-GET

请求方式为GET时，请求参数一般都直接跟在url后面，不需要参数类型。当然也可以请求参数不跟在url后面，不过比较少见。

示例接口

@HTTPRequest.route('/api/HTTPRequest2/', methods=['POST', 'GET'])  
def parameters():  
    if request.method == 'GET':  
        return "此次请求的请求方式是GET，请求参数为: name="+request.args.get('name')  
    return "请使用GET请求方式"

Part3.请求参数-POST&application/x-www-from-urlencoded

请求参数为POST时，常用的的请求参数类型(Content-Type)有application/x-www-from-urlencoded、multipart/form-data、raw、binary。

application/x-www-from-urlencoded会将表单内的数据转换为键值对，比如name=python&age = 22，这种方式只能以键值对形式发送参数，一般如果不指定content-type，默认便是application/x-www-form-urlencoded。

示例接口

@HTTPRequest.route('/api/HTTPRequest2/', methods=['POST', 'GET'])  
def parameters():  
    if request.method == 'GET':  
        return "此次请求的请求方式是GET，请求参数为: name="+request.args.get('name')  
    return "请使用GET请求方式"

    请求方式：POST

    参数值：随意写就行

运行结果

Request Headers中的Content-Type: application/x-www-form-urlencoded; charset=UTF-8，表明该请求参数类型为application/x-www-form-urlencoded。

Request Body中的POST data也是键值对形式： aaaa=EtherealBoyJIang&bbb=EtherealBoyJIang&ccc=EtherealBoyJIang


Part4.请求参数-POST&multipart/form-data

multipart/form-data会将表单的数据处理为一条消息，以标签为单元，用分隔符分开。就像这样：

POST /profile HTTP/1.1
HOST: example.com
Content-Type: multipart/form-data; boundary=example-part-boundary    
# multipart/form-data; boundary=example-part-boundary表示这个请求的的消息类型是 
# multipart-form-data，每个 part 之间的边界为 example-part-boundary。
# example-part-boundary是一段很长的字符串，如bqtFfLzQXUbOvsHrs8eOO9kIlZ30hA。

--example-part-boundary
Content-Disposition: form-data; name="username"
Content-Type: text/plain

Nicholas
# 第一个 part 的类型为 text/plain，它在表单上对应的 key 为 username，value 为 Nicholas
--example-part-boundary
Content-Disposition: form-data; name="address"
Content-Type: application/json

{
    "country": "China",
    "city": "Beijing"
}
# 第二个 part 的类型为 application/json，它在表单上对应的 key 为 address。

--example-part-boundary
Content-Disposition: form-data; name="avatar"; filename="my_avatar.jpeg"
Content-Type: image/jpeg

<binary-image data>
# 第三个 part 的数据类型为 image/jpeg，它在表单上对应的 key 为 avatar，
# 并且 part 的头部还附加了文件名相关的元数据 filename="my_avatar.jpeg。
--example-part-boundary--
# 最后面的 --example-part-boundary-- 表示整个 multipart 消息的结束。

由于这种方式可以将数据分为很多部分，它既可以上传键值对，也可以上传文件，甚至多个文件。当上传的字段是文件时，会有Content-Type来说明文件类型：Content-disposition，用来说明字段的一些信息。 每部分都是以-boundary开始，紧接着是内容描述信息，然后是回车，最后是字段具体内容(字段、文本或二进制等)。如果传输的是文件，还要包含文件名和文件类型信息。消息主体最后以-boundary-标示结束。

当使用POST方式进行请求&请求参数包含一般参数&文件时，会自动指定参数类型为multipart/form-data。

示例接口

@HTTPRequest.route('/api/HTTPRequest1/', methods=['POST'])
def upload_file():
    if 'username' not in request.form:
        return jsonify({'error': '缺少username字段'}), 400

    if 'address' not in request.form:
        return jsonify({'error': '缺少address字段'}), 400

    if 'image' not in request.files:
        return jsonify({'error': '缺少image字段'}), 400

    username = request.form['username']
    address = request.form['address']
    image_file = request.files['image']

    if image_file.filename == '':
        return jsonify({'error': '未选择文件'}), 400

    if image_file:
        filename = secure_filename(image_file.filename)
        save_path = 'C:/image/'  # 指定保存路径

        # 检查保存路径是否存在，如果不存在则创建
        if not os.path.exists(save_path):
            os.makedirs(save_path)

        # 保存文件到指定保存路径，Jmeter请求后可以到save_path指定路径查看上传的文件
        image_file.save(os.path.join(save_path, filename))

        return jsonify({
            'username': username,
            'address': address,
            'avatar_filename': filename
        }), 200

    return jsonify({'error': '未选择文件'}), 400

Part5.请求参数-POST&text/plain

raw是指可以上传任意格式的文本，如text、json、xml、html等。这里演示一下如何上传文本内容。

text/plain用于发送纯文本内容，空格转换为 “+” 加号，不对特殊字符进行编码，一般用于email之类的；

在Jmeter中发送纯文本内容，需要在消息体中输入相应内容&不在消息头管理器指定对应的content-type。会自动匹配content-type为text/plain

示例接口

@HTTPRequest.route('/api/HTTPRequest2/', methods=['POST', 'GET'])  
def parameters():  
    if request.method == 'GET':  
        return "此次请求的请求方式是GET，请求参数为: name="+request.args.get('name')  
    return "请使用GET请求方式"

示例Jmeter脚本

  • 线程组中添加HTTP 请求

    请求地址：HTTP://127.0.0.1:5000/api/HTTPRequest2/

    请求方式：POST

    消息体数据：随意（中间加空格）写就行

  • 测试计划中添加查看结果树

运行结果

Request Headers中的Content-Type: text/plain; charset=UTF-8，表明该请求参数类型为text/plain。

Request Body中的POST data就是纯文本：随意写就行。


Part6.请求参数-POST&application/json

application/json用于发送JSON字符串，也是比较常用的一种方式。

在Jmeter中发送JSON字符串，需要在消息体中输入JSON格式内容&在消息头管理器指定对应的content-type为content-type为application/json。

示例接口

@HTTPRequest.route('/api/HTTPRequest3/', methods=['POST', 'GET'])  
def httprequest3():  
    if request.method == 'POST':  
        data = request.get_json()  
        if "name" not in data:  
            return "请求参数错误"  
        return "此次请求的请求方式是GET，请求参数为: " + str(data)  
    return "请使用POST请求方式"

Part7.请求参数-POST&binary

binary对应着http请求中的Content-Type:application/octet-stream，只可以上传二进制数据，通常用来上传文件，由于没有键值，所以，一次只能上传二进制文件。

在Jmeter中发送二进制文件，需要在消息体中输入二进制数据&在消息头管理器指定对应的content-type为application/octet-stream。

文本转二进制工具

示例接口

def httprequest4():  
    if request.method != 'POST':  
        return '你的请求方式有误', 405  
    if request.content_type == 'application/octet-stream':  
        stream = request.stream  
        data = stream.read()  
        print(data)  
        try:  
            text_data = data.decode('utf-8')  
            print(text_data)  
        except UnicodeDecodeError:  
            return '解码错误，数据可能不是UTF-8编码', 400  
  
        binary_strings = text_data.split()  
  
        decimal_numbers = [int(binary, 2) for binary in binary_strings]  
  
        unicode_characters = [chr(decimal) for decimal in decimal_numbers]  
        text = ''  
        for x in unicode_characters:  
            text = text + x  
        return text, 200  
    else:  
        return '不支持的Content-Type', 415
